# TMC AI OCR PROGRAM

โปรแกรมอ่านใบสั่งซื้อ (PO) จากลูกค้าที่เป็นไฟล์ PDF (ภาพสแกน) แปลงเป็นข้อมูล
แล้วจับคู่ชื่อสินค้าเพื่อหา `tmc_code` และบันทึกเป็นไฟล์ Excel + สรุปยอดรายเดือน
ใช้งานบน **Windows 11**

---

## 1. สิ่งที่ต้องมี (ติดตั้งครั้งเดียว)

| รายการ | สถานะบนเครื่องนี้ | หมายเหตุ |
|--------|------------------|----------|
| Python 3.10+ | ✅ พบแล้ว (3.14) | |
| Tesseract OCR | ✅ พบที่ `%LOCALAPPDATA%\Programs\Tesseract-OCR` | โปรแกรมตั้งค่า path ให้อัตโนมัติ |
| Poppler (แปลง PDF→ภาพ) | ✅ ใช้ที่มากับโปรเจกต์ `..\poppler\Library\bin` | |
| ไลบรารี Python | ต้องติดตั้ง | ดับเบิลคลิก `install.bat` |

**ติดตั้ง:** ดับเบิลคลิก **`install.bat`** (รันครั้งเดียว)

> **ภาษาไทย**: ติดตั้งชุดภาษาไทย (`tha.traineddata`) ไว้แล้ว และตั้งค่า OCR เป็น `tha+eng`
> เพราะ PO บางใบมีชื่อ/คำอธิบายสินค้าเป็นภาษาไทย (ถ้า PO ของลูกค้าเป็นอังกฤษล้วน จะตั้งเป็น
> `eng` ในแท็บตั้งค่าเพื่อให้เร็ว/สะอาดขึ้นก็ได้)

## 2. วิธีเปิดโปรแกรม

ดับเบิลคลิก **`run.bat`**  (หรือสั่ง `python main.py`)

## 3. โครงสร้างโฟลเดอร์ที่โปรแกรมใช้

```
TMC INVOICE INPUT\            ← "โฟลเดอร์หลัก" (ตั้งค่าได้ในแท็บตั้งค่า)
├── DATA\คลังสินค้า.xlsx      ← แหล่ง dropdown stock_group_code
├── CMT\                      ← 1 โฟลเดอร์ = 1 ลูกค้า
│   ├── PO FROM CUSTOMER\     ← ไฟล์ PO (.pdf) ของลูกค้า
│   ├── PRODUCT DETAIL\Product Details.xlsx   ← ตารางจับคู่ชื่อสินค้า → tmc_code
│   └── INVOICE FILE\         ← โปรแกรมบันทึกผลลัพธ์ Excel ไว้ที่นี่
└── Ogihara\ ...
```

## 4. ขั้นตอนการใช้งาน

0. **แท็บ 1 – ลงทะเบียนลูกค้า** (ทำครั้งแรกต่อลูกค้า 1 ครั้ง)
   - **ชื่อสินค้าของลูกค้าแต่ละรายไม่เหมือนกัน** จึงต้องมีไฟล์ `tmc_code` (Product Details.xlsx)
     ของลูกค้าแต่ละรายแยกกัน
   - กด **➕ สร้างลูกค้า** → โปรแกรมสร้างโฟลเดอร์ให้ (PO / PRODUCT DETAIL / INVOICE FILE)
   - เลือกลูกค้าในตาราง → **📎 นำเข้าไฟล์ tmc_code** เพื่อแนบไฟล์ mapping ของลูกค้ารายนั้น
     (โปรแกรมตรวจสอบว่ามีคอลัมน์ `ชื่อสินค้าลูกค้า` และ `tmc_code` ให้อัตโนมัติ)
   - ตารางแสดงสถานะ: มีไฟล์ tmc_code หรือยัง, จำนวนสินค้าใน mapping, จำนวนไฟล์ PO, เรียนรู้ตำแหน่งแล้วหรือยัง

1. **แท็บ 2 – ประมวลผล PO**
   - เลือก **ลูกค้า** (โปรแกรมจะโหลดไฟล์ PO และตาราง Product Details ให้อัตโนมัติ)
   - **📥 นำเข้าไฟล์ PO เข้าโฟลเดอร์ลูกค้า**: เลือกไฟล์ PDF จากที่ไหนก็ได้ โปรแกรมคัดลอกเข้าโฟลเดอร์
     PO ของลูกค้าให้ (สร้างโฟลเดอร์อัตโนมัติ ถ้าไฟล์ซ้ำจะข้าม)
   - กด **▶ อ่านเอกสาร (OCR)** → โปรแกรมอ่านทุกไฟล์ในรายการ
   - คลิกไฟล์ในรายการเพื่อดู/แก้ไขผลลัพธ์ในตาราง
   - **ตรวจทานและแก้ไข**: `tmc_code` (จับคู่อัตโนมัติ), เลือก `stock_group_code` จาก dropdown,
     แก้ `จำนวน`/`ราคา` ให้ตรงกับเอกสาร (ดูคอลัมน์ "ชื่อสินค้า (OCR)" ประกอบ)
   - คอลัมน์ **"ชื่อสินค้าที่จับคู่ (TMC)"** แสดงชื่อสินค้าจริงของ `tmc_code` ที่เลือก — เปลี่ยนรหัสแล้ว
     ชื่ออัปเดตทันที ช่วยตรวจว่าจับคู่ถูกหรือไม่
   - ช่อง **`tmc_code` ค้นด้วยชื่อสินค้าได้** (พิมพ์ชื่อไทย/อังกฤษบางส่วน เช่น "NOZZLE" หรือ "บู๊ช"
     แล้วเลือก ระบบเติมรหัสให้) — เหมาะเวลาที่ OCR อ่านชื่อเพี้ยน
   - ปุ่ม **ลบแถวที่เลือก / เพิ่มแถว / คำนวณยอดรวมใหม่** ตามต้องการ
   - กด **🧾 รวมทุกไฟล์ในโฟลเดอร์เป็น Excel เดียว**: นำรายการสินค้าจาก **ทุก PO ที่อ่านแล้ว** มาต่อกัน
     ในชีต `Invoice` เดียว (คอลัมน์เดิม `tmc_code|stock_group_code|qty|price`) + ชีต `Summary` แยกรายใบ
   - หรือกด **💾 บันทึกไฟล์นี้เป็น Excel + เก็บสรุปรายเดือน** (เฉพาะ PO ที่เลือก)
     - **เลือกที่เก็บไฟล์ได้เอง** (ค่าเริ่มต้นคือโฟลเดอร์ `INVOICE FILE\` ของลูกค้า)
     - ชื่อไฟล์ตามรูปแบบ **`PO Number_วันที่สร้าง.xlsx`** (เช่น `69110610_20260701.xlsx`)
     - ได้ไฟล์ Excel คอลัมน์ `tmc_code | stock_group_code | qty | price`
     - บันทึกสรุป (ลูกค้า, เลข PO, จำนวนรายการ, ยอดรวม, VAT, รวมทั้งสิ้น) ลงฐานข้อมูลของโปรแกรม

2. **แท็บ 3 – สรุปรายเดือน**
   - กรองตาม **ลูกค้า** และ **ช่วงวันที่** ได้
   - **ลบรายการที่ผิดพลาด** ได้ (เลือกแถว → ปุ่มลบ)

3. **แท็บ 4 – ตั้งค่า** — แก้ path ต่าง ๆ, DPI, เกณฑ์จับคู่ชื่อ (fuzzy)

## 5. การ "เรียนรู้ตำแหน่ง"

ลูกค้าแต่ละรายเลย์เอาต์ต่างกันมาก (CMT มีเส้นตาราง / NIPPON ไม่มีเส้นและชื่ออยู่บรรทัด 2 /
TSGP มี 8 คอลัมน์) จึงมี 2 วิธีสอนตำแหน่ง — เก็บเป็นโปรไฟล์ `templates\<ลูกค้า>.json`

1. **อัตโนมัติ** — ลูกค้าใหม่ที่ PO มีเส้นตารางชัด โปรแกรมตรวจจับให้เอง
   (กด **↻ เรียนรู้ตำแหน่งอัตโนมัติ** เพื่อบันทึกซ้ำได้)
2. **สอนเองแบบลากกรอบ (แนะนำเมื่ออ่านผิด)** — กด **🖼️ สอนตำแหน่ง (ลากกรอบ)**
   - จะเปิดภาพเอกสาร → กดปุ่มเลือกช่อง (ลำดับ / ชื่อสินค้า / จำนวน / ราคา / จำนวนเงิน)
     แล้ว **ลากเมาส์ครอบคอลัมน์นั้น** ทำให้ครบ
   - ติ๊ก **“ชื่อสินค้าอยู่บรรทัดถัดไป”** ถ้าชื่อจริงอยู่บรรทัด 2 (เช่น NIPPON)
   - เลือก **รูปแบบตัวเลข** (ทศนิยม 2/3 ตำแหน่ง หรือปกติ)
   - กด **บันทึกโปรไฟล์** → กด “อ่านเอกสาร (OCR)” ใหม่

โปรไฟล์รองรับ: `columns` (กล่องคอลัมน์), `number_mode` (fixed2/fixed3/decimal),
`name_below`, `data_top_frac`/`bottom_frac` (ตัดหัว/ท้ายกระดาษ), อ่านครบทุกหน้าอัตโนมัติ

## 6. ข้อค้นพบสำคัญ & คำแนะนำ

1. **โปรแกรมเวอร์ชันก่อนหน้าอ่านผิดคอลัมน์** — ไฟล์ `CMT\INVOICE FILE\Review_Notes...txt`
   แสดงว่าเดิมไปอ่านคอลัมน์ *รหัสสินค้า (PRODUCT CODE)* ซึ่งเป็นรหัสภายในของลูกค้า จึงจับคู่ไม่ได้
   (`no_match` ทั้งหมด). เวอร์ชันนี้ใช้คอลัมน์ **รายการ/DESCRIPTION** (เลขพาร์ทจริง เช่น
   `DNMG150612-HN-GK1120`) มาจับคู่ + ใช้ **fuzzy matching** เพื่อทนต่อความเพี้ยนของ OCR → จับคู่ได้แม่นขึ้นมาก

2. **สแกนแบบดอทเมทริกซ์จางมาก** — Tesseract มักอ่านเลขทศนิยมหาย (`100.00`→`10000`).
   โปรแกรมแก้ด้วยกฎ "ทศนิยม 2 ตำแหน่งเสมอ" (เก็บเฉพาะตัวเลข ÷ 100) ทำให้ยอดเงินถูกต้อง
   **แต่รหัส/ตัวเลขบางช่องยังผิดได้** จึงออกแบบให้ "คนตรวจแก้ในตาราง" ก่อนบันทึกเสมอ

3. **ถ้าต้องการความแม่นสูงขึ้นอีกมาก** แนะนำอัปเกรดเป็น **AI Vision OCR** (เช่นส่งภาพหน้า PO ให้
   โมเดลภาพของ Claude/Google Vision อ่าน) จะอ่านสแกนจาง ๆ ได้ดีกว่า Tesseract อย่างเห็นได้ชัด
   — โครงสร้างโค้ด (`app/ocr.py`) แยกส่วนไว้ให้เปลี่ยน engine ได้ในอนาคต

4. **คุณภาพไฟล์ข้อมูลต้นทาง** — พบว่าบางแถวใน `Product Details.xlsx` ใส่ค่า `tmc_code` ผิด
   (เช่นแถว TNMG160408-GF-GP1115 ช่อง tmc_code เป็นชื่อสินค้า ไม่ใช่รหัส `T0…`)
   แนะนำให้ไล่ตรวจไฟล์นี้ให้สะอาด จะช่วยให้จับคู่ถูกต้องอัตโนมัติมากขึ้น

5. **เคล็ดลับสแกนให้อ่านแม่น**: สแกน 300 DPI ขึ้นไป, วางกระดาษตรง (ไม่เอียง),
   ตั้งความเข้ม (contrast) ให้ตัวอักษรชัด, ลอกกระดาษก็อปปี้แผ่นที่คมที่สุด

6. **สำรองข้อมูล** — สรุปรายเดือนเก็บใน `TMC_OCR\tmc_ocr.db` (SQLite) ควรสำรองไฟล์นี้เป็นระยะ

```
│   └── ui\                 หน้าจอ (PySide6): process / summary / settings
```
